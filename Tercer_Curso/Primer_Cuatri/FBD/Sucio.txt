Definición de modelo de datos
-----------------------------------------

Inventado por Edgar F.Codd (matemático).

Un modelo logico de la base de datos es el esquema conceptual retocado para que pueda ser implementado en sistema gestor de base de datos.

Este model opermite una implementacion muy solida, estando muy bien formalizado desde el punto de vista matemático, luego las empresas saben muy bien que tienen que hacer.

Un modelo de datos e s un mecanismo formal para representar y manipular la indormacion de manera general y sistematica que debe contener:
    - Notacion para describir datos
    - Notacion para describir operaciones
    - Nrotacion para describir restricciones de integridad

Modelo de datos relacional
--------------------------------------

Representamos los dato ene forma de tablas que son relaciones(nombre formal)., en realidad este es el nombre que da Codd.

Para Codd, la representacion de las n-uplas que haya en la tabla de forma conjunta se representan cada dato (columna) con un dominio especifico luego un n-upla será un subconjunto del producto cartesiano de todos los dominios donde tenemos una relacion n-aria.

Cada tabla debe disponer de un nombre único, no puede haber dos relaciones.

Hay tras ámbitos del modelo de datos relacional:

    - Consulta y mainuplacion -> operadores que permite hacer consultas mediante la definición matematica de los mismos creando el algebra relacional permitiendo operaciones sobre tablas. Las consultas serán operaciones que resultan tables mediante que la composicion obtneer una tabla que de todo el resultado buscado.

    - Integridad .> los datos deben menatener su consistencia
    - Estructuras -> relaciones

Propiedades de una estructura de datos:

- Condicion de normalizacion: todos los valores de los atributos de una relación deben ser atómicos, es decir no son estructurados <-> lo consideramos como un todo
    Si se cumple la condicion se dice que esta en primera forma normal (son condiciones de calidad que se definen formalmente y nos dicen como de bueno es un esquema relacional) Estas formas normales son sellos de acalidad para saber como de buena es un esquema de datos.
    la 5ª forma normal sginifaica que el diseño relacional no se puede mejorar

- Consecuencias de la primera forma normal:
    - No hay valores de tripo conjutno, registro o tabla poues son atomicos

- PRoblema -> todas las representaciones son estensibvas <-> no se puede representar direcrtamente informacion del tipo "una unica instancia sera repetida con distintos atributos". -> da facilidad y consulta de datos

Cuando definimos una tabla no hay orden en las columnas ni siqueira en las filas -> luego dos instancias con distinto orden son la misma instancia. -> debo acceder a un dato por su valor y no pos su posicion. De esta manera las clasves priamrias son importantes pues tenemos un valor por el que podemos buscar. Buscar por criterio posicional implica perder la independencia de los datos pues un cambio el orden la scolumnas rompemos el sistema. Por definicion de conjunto no habra tuplas dupicadas y tendremos la conjuncion de todas las tuplas como clave primaria consiguiendo el acceso mas restrictivo posible.


Valor nulo: Cuando no concemos el valor de un atrburto para una deteminada tupla le asociamos un valor especial (valor nulo) -> este valor debe aparecer en todos los dominios y todos los dominios tendrán uno que no deberán ser el mismo; de hecho, aunque este disponible en el domino de la clave priamria no podrá tomarse el valor por su definicion.

- Un valor nulo puede ser un valor que desconozco
- Puede ser un atributo no aplicable.

Esto puede producir confusiones  luego buena parte del esfuerzo será buscar que no se usen muchos valores nulos en la consulta de datos.

Restricciones de integridad
---------------------------------

La integridad es lo relativo a definir reglas que permitan garantizar la correccion semantica de la base de datos. Dos tiposde reglas:
    - Específica -> relacionadas con definir dominios
    - Pripias del papel de los atributos-> restricciones de sentido comun por loq ue represente el atributo. La restriccion de que una clave primaria no sea nula es de este tipo.
